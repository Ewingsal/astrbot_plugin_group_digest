# astrbot_plugin_group_digest

这是一个群聊兴趣日报与主动互动插件。  
基于 AstrBot 实现群聊消息总结、群热门话题分析、成员兴趣摘要与日报生成、通过LLM 建议astrbot定时主动发言。

## 功能简介

这个插件主要解决两类问题：

1. **命令式日报**
   - `/group_digest`：生成昨天全天的群聊兴趣日报
   - `/group_digest_today`：生成今天 00:00 到当前时刻的群聊兴趣日报
   - `/group_digest_debug_today`：输出今日底层统计信息

2. **定时主动互动**
   - 支持每天固定时间自动触发
   - 按群独立分析当天聊天内容
   - 不发送整份日报，只发送一条适合当前群聊语境的主动发言

## 核心特点

- **按群隔离分析**  
  每个群的消息、日报和主动发言都独立处理，不会串群。

- **规则统计 + LLM 语义分析**  
  规则逻辑负责发言数、活跃成员、统计范围等确定性信息；  
  LLM 负责热门话题、成员兴趣摘要、整体总结和建议 Bot 主动发言。

- **多群定时主动发言**  
  到达设定时间后，插件会对已记录的群分别生成分析结果，并向对应群发送各自的建议发言。

- **日报缓存机制**  
  同一天、同一群、同一模式下，如果没有新的有效消息，会直接复用缓存日报，减少重复分析开销。

- **可配置分析模型**  
  可在插件设置中指定用于语义分析的模型；留空时默认复用 AstrBot 当前会话模型。

## 日报内容

生成的日报通常包含：

- 统计日期与统计范围
- 总发言数
- 参与成员数
- 活跃成员排行
- 热门话题
- 成员兴趣摘要
- 整体总结
- 建议 Bot 主动发言

## 设计思路

插件整体采用“两层处理”思路：

### 1. 规则层
负责确定性统计，例如：

- 总发言数
- 活跃成员排行
- 统计时间窗口
- 按群隔离的数据组织

### 2. 语义层
基于 LLM 对群聊内容进行理解，生成：

- 热门话题
- 成员兴趣摘要
- 整体总结
- 建议 Bot 主动发言

这样做的好处是：

- 数字统计更稳定
- 语义分析更自然
- 更适合“日报总结 + 主动互动”这种场景

## 定时主动发言

插件支持每天固定时间自动发言。

行为规则如下：

- 仅支持群聊
- 每个群独立生成自己的分析结果
- 每个群只会收到属于自己的主动发言
- 主动发言内容来自日报分析中的 `suggested_bot_reply`
- 支持白名单群配置

这个能力适合让 Bot 在固定时间点自然参与群聊，而不仅仅是被动响应命令。

## 缓存机制

为了减少重复分析开销，插件内置了日报缓存：

- 日报成功生成后会写入缓存
- 命令触发和定时任务触发都会复用同一套缓存机制
- 如果没有新的有效消息，则直接命中缓存
- 如果有新的有效消息，则重新生成并刷新缓存

## 支持的配置项

插件支持通过 AstrBot 配置项进行调整，主要包括：

- 是否启用 LLM 语义分析
- 指定分析模型 provider
- LLM 分析提示词模板
- 最大分析消息条数
- 是否仅在统计模式下回退
- 是否启用定时主动发言
- 定时触发时间
- 调度时区
- 是否启用群白名单
- 白名单群列表

## 存储说明

插件运行时数据默认存放在 AstrBot `data` 目录下，包括：

- 群消息归档
- 群会话标识映射
- 日报缓存

## 适用场景

这个插件适合：

- 想快速了解某个群过去一天主要聊了什么
- 想观察群成员最近关注的话题
- 想让 Bot 在固定时间自然参与讨论
- 想把群聊总结和主动互动做成自动化能力

## 项目定位

这不是一个单纯的“关键词统计插件”，而是一个面向群聊场景的：

**日报生成 + 语义分析 + 主动互动**

一体化插件。
